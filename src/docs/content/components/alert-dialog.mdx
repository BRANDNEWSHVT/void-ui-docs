# Alert Dialog

A modal dialog that interrupts the user with important content and expects a response. Use for destructive actions that require explicit confirmation.

## Usage

### Destructive Action

The most common use case is confirming destructive actions like deleting data.

<ComponentPreview
  code={`<AlertDialog>
  <AlertDialogTrigger>
    <Button variant="outline-danger">Remove Item</Button>
  </AlertDialogTrigger>
  <AlertDialogContent>
    <AlertDialogTitle>Remove this item?</AlertDialogTitle>
    <AlertDialogDescription>
      This item will be moved to trash and permanently deleted after 30 days.
    </AlertDialogDescription>
    <AlertDialogFooter>
      <AlertDialogCancel>Keep Item</AlertDialogCancel>
      <AlertDialogAction>Remove</AlertDialogAction>
    </AlertDialogFooter>
  </AlertDialogContent>
</AlertDialog>`}
>
  <AlertDialog>
    <AlertDialogTrigger>
      <Button variant="outline-danger">Remove Item</Button>
    </AlertDialogTrigger>

      <AlertDialogDescription>
        This item will be moved to trash and permanently deleted after 30 days.

      <AlertDialogFooter>
        <AlertDialogCancel>Keep Item</AlertDialogCancel>
        <AlertDialogAction>Remove</AlertDialogAction>

### AlertDialogTitle

The accessible title for the dialog. Required for screen readers.

### AlertDialogDescription

The accessible description for the dialog.

### AlertDialogFooter

Container for action buttons. Displays buttons in a row with proper spacing.

### AlertDialogCancel

Secondary button that closes the dialog without action.

### AlertDialogAction

Primary button for confirming the action. Styled with `danger` variant by default.

## Accessibility

- **Focus trap** - Focus is contained within the dialog while open
- **Escape key** - Closes the dialog
- **Click outside** - Does not close (requires explicit action)
- **Screen readers** - Title and description are announced
- **Focus return** - Focus returns to trigger when closed
