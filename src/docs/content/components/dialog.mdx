# Dialog

A modal dialog that interrupts the user with important content. Features smooth animations and backdrop blur.

## API Reference

### Dialog

The root component that manages dialog state.

### DialogTrigger

The element that opens the dialog when clicked.

### DialogContent

The modal container with backdrop and animations.

<PropsTable
  props={[
    {
      name: 'showClose',
      type: 'boolean',
      default: 'true',
      description: 'Show the X close button in the corner.',
    },
    {
      name: 'className',
      type: 'string',
      description: 'Additional CSS classes (e.g., max-w-sm for smaller dialogs).',
    },
  ]}
/>

### DialogHeader

Container for title and description.

### DialogTitle

The dialog heading.

### DialogDescription

Secondary text describing the dialog purpose.

### DialogFooter

Container for action buttons, aligned to the right.

### DialogClose

Wraps elements that should close the dialog when clicked.

## Accessibility

- Focus is trapped within the dialog when open
- `Escape` key closes the dialog
- Clicking the backdrop closes the dialog
- Focus returns to the trigger when closed
- Proper ARIA attributes for screen readers
