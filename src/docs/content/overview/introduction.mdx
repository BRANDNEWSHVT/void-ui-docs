# Introduction

Void UI is a personal React component library built on [Base UI](https://base-ui.com) and styled with [Tailwind CSS](https://tailwindcss.com). Dark-first design with a modern aesthetic for web applications.

> **Experimental**: This library is still in active development and primarily built for my personal projects. APIs may change without notice. Feel free to use it, but be aware of potential breaking changes.

## Why This Exists

I created this documentation mainly to help myself remember how to use these components across my projects. If you find it useful, feel free to use it in your own projects.

## Features

- **Accessible** - Built on Base UI with WAI-ARIA compliance
- **Customizable** - CSS variables and semantic color tokens
- **TypeScript** - Full type definitions
- **Phosphor Icons** - Integrated icon library

## Design Tokens

Void UI uses CSS custom properties (variables) for theming. Override these in your CSS to customize the look and feel.

## Customization

### Overriding Colors

Create a custom theme by overriding CSS variables in your global stylesheet:

<CodeBlock
  code={`/* Custom brand colors */
:root {
  --void-primary: #8b5cf6;
  --void-success: #10b981;
}

/* Light mode overrides */
[data-theme="light"] {
  --void-bg: #ffffff;
  --void-surface: #f4f4f5;
  --void-text: #18181b;
  --void-muted: #71717a;
  --void-border: #e4e4e7;
}`}
  language="css"
  filename="app.css"
/>

### Component-Level Customization

All components accept a `className` prop for Tailwind overrides:

<CodeBlock
  code={`<Button className="bg-purple-500 hover:bg-purple-600">
  Custom Button
</Button>

<Card className="border-2 border-dashed">
  Custom Card
</Card>`}
  language="tsx"
/>

### Extending with Variants

Use the `cn()` utility to conditionally apply classes:

<CodeBlock
  code={`import { cn } from '@/lib/utils'

<Badge className={cn(
  isNew && 'animate-pulse',
  isPriority && 'ring-2 ring-offset-2'
)}>
  Label
</Badge>`}
  language="tsx"
/>
